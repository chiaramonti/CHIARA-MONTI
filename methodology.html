Methodology

This second section is devoted to a detailed explanation of the methodology adopted throughout the project.

As an initial step, we identified a clearly defined area of focus that aligned with our research interests. For the purposes of this study, we chose to concentrate on Archaeological Sites (as already mentioned in the home page), given their historical significance and relevance within the broader context of cultural heritage.

Accordingly, we began our research within the ArCo dataset by designing and executing the following SPARQL queries.

SPARQL Query Result 
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX arco: <https://w3id.org/arco/ontology/arco/>

SELECT DISTINCT ?cp ?label ?depiction
WHERE {
  ?cp rdfs:label ?label ;
      rdf:type arco:CulturalProperty .
  FILTER(REGEX(?label , "parco archeologico" , "i"))
  OPTIONAL { ?cp foaf:depiction ?depiction }
}
ORDER BY (?label)
LIMIT 100
The query retrieves up to 100 cultural properties that are labeled as “parco archeologico”, along with their labels and any available images, sorted alphabetically by name.

However, the results of this query were too general for our investigation, so we decided to formulate another query that investigated the class ArchaeologicalParks.

Subsequently, in order to enrich the data related to the archaeological parks we had selected, we decided to consult ChatGPT, which provided us with the following response. To formulate this question, we employed the zero-shot prompting technique.

By reviewing the results provided by ChatGPT and comparing them with the information already available in the ArCo database regarding the archaeological parks we selected, we have decided to add three new triples—currently missing—for each park.

The first triple concerns the precise geographical coordinates of the selected archaeological parks, expressed in terms of latitude and longitude. For this, we will use the predicate geo:lat/geo:long, already present in ArCo.

The second triple refers to the conservation status of the parks, a fundamental aspect in this context. We will introduce new triples using the ArCo predicate a-dd:hasConservationStatus.

The third triple we propose involves introducing links to equivalent entities in external datasets. We have chosen to work with Wikidata. This new triple will use the predicate owl:sameAs, which is also already included in ArCo.
